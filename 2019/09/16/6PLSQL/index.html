数据完整性的约束、事务、视图

2019/9/3 19:45:36

数据完整性的约束(重中之重)

回顾学过的约束

主键约束(primary key) - 要求该字段的数值唯一且不能为空；
唯一约束(unique)      - 要求该字段的数值唯一但可以为空；
非空约束(not null)    - 要求该字段的数值不能为空；
检查约束(check)       - 要求该字段的数值必须满足指定的格式或范围等；

外键约束

当本表中某一字段的数值取决于另外一张表的主键时，此字段就需要加上外键约束。
也就是依赖于外部的主键，当前表叫做子表，所依赖的表叫做主表

id  student表 主键   - 主表
studentid  result表  - 子表

事务(重中之重、难点、笔试面试)

事务的由来

-- 使用图形化工具向年级表中插入数据,此插入操作实际上只是把数据放到内存中
insert into grade values(7, '黑铁');
-- 没有进行提交时，命令行工具无法进行插入操作，因为图形化工具的流程没有走完
-- 类似于数据库被锁住，直到图形化工具的流程走完才能解锁
-- 这个图形化工具执行的流程 叫做 一个事务
-- 使用commit实现数据的提交，才会将数据由内存插入到数据库的表空间中
commit;
-- 查询年级表中的所有信息
select * from grade;
注意：
在以后的开发中只要使用到dml语句一定要记得commit才会提交到数据库中

视图(重中之重)

视图的由来

对于同一张表中的数据来说，不同的人群可能关注不同的数据，若将不同的数据放在不同的表中会造成表空间的浪费，此时可以使用视图来满足该需求。
视图本质上就是用于展示单个表中部分字段或者多个表中综合字段的虚拟表。

视图的创建和删除

(1)创建视图的语法格式
   create view vw_视图名
   as
   select查询语句;
(2)删除视图的语法格式
   drop view vw_视图名;
如：
   drop view vw_girls;
